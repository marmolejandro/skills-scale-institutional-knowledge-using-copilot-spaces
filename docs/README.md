## Project Management Process Summary

OctoAcme follows an iterative, outcome-driven approach to project management grounded in the following principles:

- **Customer-first**: Prioritize customer value and usability in all decisions
- **Iterative delivery**: Deliver small, testable increments rather than big-bang releases
- **Clear ownership**: Each project has named owners (Project Manager and Product Lead) with defined accountability
- **Data-informed decisions**: Measure impact and iterate based on evidence
- **Psychological safety**: Encourage feedback, learning, and continuous improvement

### Core Lifecycle

Our project lifecycle consists of five key phases:

1. **Initiation** — Define the problem statement, stakeholders, success metrics, and get sponsor approval to proceed to planning
2. **Planning** — Break work into shippable increments, estimate scope, define acceptance criteria, and map dependencies and milestones
3. **Execution** — Build, test, review, and iterate using a board-based workflow and regular ceremonies (standup, demo, retrospective)
4. **Release** — Deploy to production following standardized checklists and playbooks to minimize risk and maximize observability
5. **Close & Retrospective** — Capture learnings, document action items, and feed improvements back into future projects

### Key Roles

- **Project Manager (PM)**: Coordinates delivery, manages schedules, risks, and communications
- **Product Manager (PdM)**: Defines outcomes, prioritizes the backlog, and measures success
- **Developers**: Implement features, collaborate on design and testability
- **QA/Testing**: Validate quality and acceptance criteria
- **Stakeholders**: Provide inputs and approvals

### Communication Cadence

- **Daily**: Team standups (15 min) to discuss progress, blockers, and dependencies
- **Weekly**: PM + PdM alignment and delivery team syncs
- **Monthly**: Stakeholder updates and milestone reviews
- **Ad-hoc**: Escalations and risk discussions as needed

## Getting Started

**For new team members:**
1. Start with [Project Management Overview](octoacme-project-management-overview.md) to understand the big picture
2. Review [Roles & Personas](octoacme-roles-and-personas.md) to understand your role and responsibilities
3. Explore the phase-specific docs as you encounter each stage of a project

**For active projects:**
- Use the phase-specific docs (Initiation, Planning, Execution, Release) as checklists and playbooks
- Keep the Project One-pager and Risk Register updated
- Reference templates and examples throughout the docs

**For continuous improvement:**
- Review [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md) after each milestone or release
- Use the feedback loop to suggest updates to these docs via [Process Doc Update issues](.github/ISSUE_TEMPLATE/add-update-content-to-process-docs.yml)
